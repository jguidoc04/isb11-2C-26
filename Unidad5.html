01. Arquitecturas
// Von Neumann vs. Harvard — diferencias clave
VON NEUMANN — 1945

Memoria unificada

Datos e instrucciones comparten el mismo bus y memoria. Arquitectura de la mayoría de PCs.

  • Memoria única para datos e instrucciones
  • Bus compartido — solo una operación a la vez
  • Simple de implementar, menor costo
  • Cuello de botella de Von Neumann
  • Usado en: PCs, laptops, servidores
HARVARD — MODIFICADA

Memorias separadas

Datos e instrucciones usan buses y memorias separados. Mayor velocidad en paralelismo.

  • Memorias separadas — datos e instrucciones
  • Buses paralelos — acceso simultáneo
  • Sin cuello de botella, mayor rendimiento
  • Mayor costo y complejidad
  • Usado en: microcontroladores, DSPs, GPUs
CARACTERÍSTICA VON NEUMANN HARVARD
Memoria Unificada (datos + instr.) Separada por función
Buses Uno compartido Múltiples (paralelos)
Velocidad Limitada por bus Mayor rendimiento
Costo/Complejidad Bajo Mayor
Cuello de botella Sí (Von Neumann) No
Usos típicos PC, servidores MCU, DSP, GPU, caché CPU
02. Unidades principales
// Componentes funcionales de una computadora — haz clic para más detalle
⚙️

Unidad de Control (UC)

Dirige el flujo de datos e instrucciones en la CPU. El "director de orquesta" del sistema.

➕

ALU — Aritmético-Lógica

Ejecuta operaciones matemáticas y lógicas (suma, resta, AND, OR, comparaciones).

💾

Unidad de Memoria

Almacena datos e instrucciones. Incluye RAM, ROM, caché y registros internos.

🔌

E/S — Entrada / Salida

Interfaz con el mundo exterior. Teclado, pantalla, disco, red — todo pasa por aquí.

🚌

Bus del Sistema

Canal de comunicación entre todos los componentes. Datos, direcciones y control.

📋

Registros del CPU

Memoria ultra-rápida interna del procesador. PC, IR, MAR, MDR, ACC, FLAGS.

⚙️ Unidad de Control (UC)

FUNCIÓN PRINCIPAL
Decodificar y controlar
  • Lee instrucciones de memoria
  • Las decodifica (¿qué operación?)
  • Genera señales de control
  • Coordina ALU, registros y buses
  • Maneja el ciclo fetch-decode-execute
COMPONENTES INTERNOS
IR · PC · Decodificador · Secuenciador · Generador de señales
IMPLEMENTACIÓN
Cableada (rápida) o Microprogramada (flexible)

➕ Unidad Aritmético-Lógica (ALU)

OPERACIONES ARITMÉTICAS
Suma · Resta · Multiplicación · División
OPERACIONES LÓGICAS
AND · OR · NOT · XOR · Desplazamiento
ENTRADAS / SALIDAS
2 operandos (A, B) + código de op → Resultado + Flags (zero, carry, overflow, sign)
VELOCIDAD
Operación en 1–4 ciclos de reloj

💾 Jerarquía de Memoria

REGISTROS
~1 ciclo · bytes · en el CPU
CACHÉ L1/L2/L3
4–40 ciclos · KB/MB · en chip
RAM (Memoria principal)
~200 ciclos · GB · volátil
SSD / HDD (Almacenamiento)
ms / TB · no volátil

🔌 Entrada / Salida (E/S)

DISPOSITIVOS DE ENTRADA
Teclado · Ratón · Escáner · Cámara · Micrófono · Sensores
DISPOSITIVOS DE SALIDA
Monitor · Impresora · Altavoces · Proyector · Actuadores
MÉTODOS DE TRANSFERENCIA
Polling · Interrupciones · DMA (Acceso directo a memoria)
INTERFACES
USB · PCIe · SATA · HDMI · Bluetooth · Ethernet

🚌 Bus del Sistema

BUS DE DATOS
Transfiere datos entre CPU, memoria y E/S
BUS DE DIRECCIONES
Especifica la dirección de memoria a acceder
BUS DE CONTROL
Señales de lectura/escritura, interrupciones, reloj
ANCHO TÍPICO
64 bits (datos) · 48 bits (dirección) en CPU modernos

📋 Registros del CPU

PC — Program Counter
Dirección de la próxima instrucción
IR — Instruction Register
Instrucción en ejecución actual
MAR — Memory Address
Dirección de memoria a leer/escribir
MDR — Memory Data
Dato leído de / a escribir en memoria
ACC — Acumulador
Resultado de operaciones ALU
FLAGS / PSW
Zero · Carry · Overflow · Sign
